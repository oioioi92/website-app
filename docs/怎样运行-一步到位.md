# 怎样运行 — 一步到位

分两种：**本地 Windows 开发**、**服务器上运行**。任选其一，按顺序执行即可。

---

## 一、本地 Windows 运行（开发）

在 **PowerShell** 里，**进入项目根目录** `C:\Users\user\Desktop\Website-new` 后执行。

### 方式 A：主站 + Live Chat 一起跑（推荐，一次搞定）

```powershell
cd C:\Users\user\Desktop\Website-new
npm install
npm run dev
```

**一次命令**同时启动主站和 chat-server，不再出现「Live Chat 服务未连接：fetch failed」。同一终端会看到主站（main）和聊天服务（chat）的日志，Ctrl+C 会同时停掉两者。首次运行前若未安装过 chat-server 依赖，请先执行：`cd services/chat-server; npm install`。

浏览器打开：前台 http://localhost:3000 ，后台 http://localhost:3000/admin/login ，Live Chat 管理 http://localhost:3000/admin/chat 。

**说明**：每次执行 `npm run dev` 时会先自动释放 3000 端口并删除 `.next\dev\lock`。若只想跑主站、不跑 Live Chat，可执行 `npm run dev:main`。

**若出现 Prisma「EPERM: operation not permitted」**：关掉 Cursor，新开 PowerShell 执行 `cd 项目根目录` 再 `.\scripts\prisma-generate-fix.ps1`，然后重新打开 Cursor 并运行 `npm run dev`。

---

### 方式 B：主站 + Live Chat 开两个窗口

```powershell
cd C:\Users\user\Desktop\Website-new
powershell -ExecutionPolicy Bypass -File .\scripts\dev-with-chat.ps1
```

脚本会开两个新窗口分别跑主站和 chat-server，并自动打开前台/后台页面。

---

### 方式 C：只跑主站（不跑 Live Chat）

```powershell
npm run dev:main
```

后台 Live Chat 页面会提示「服务未连接」，属预期。

## 二、服务器上运行（部署后）

SSH 登录服务器后，在 **bash** 里执行。项目路径按你实际来（例如 `/root/website-new`）。

### 1. 首次或更新代码后：安装 + 构建 + 启动

```bash
cd /root/website-new
npm install
node scripts/prisma-generate-auto.cjs
node scripts/prisma-db-push-auto.cjs
npm run build
pm2 start node_modules/next/dist/bin/next --name website-phase2 -- start -p 3000
pm2 save
```

### 2. 启动 Live Chat（chat-server）

```bash
cd /root/website-new/services/chat-server
npm ci
npm run prisma:generate
npm run prisma:migrate 2>/dev/null || npm run migrate:sqlite 2>/dev/null || true
pm2 start npm --name chat-server -- start
pm2 save
```

### 3. 以后只重启（代码未改、只重启进程）

```bash
pm2 restart website-phase2
pm2 restart chat-server
pm2 save
```

### 4. 查看状态 / 日志

```bash
pm2 status
pm2 logs website-phase2 --lines 50
pm2 logs chat-server --lines 50
```

## 三、从本机一键部署到服务器（上传并运行）

在 **本机 PowerShell**、项目根目录下执行（会提示输入 SSH 密码两次）：

```powershell
cd C:\Users\user\Desktop\Website-new
.\scripts\deploy-one-click.ps1 -HostAddr "root@68.183.227.172"
```

若要连同 chat-server 一起部署：

```powershell
$env:DEPLOY_CHAT_SERVER="1"; .\scripts\deploy-one-click.ps1 -HostAddr "root@68.183.227.172"
```

部署完成后，服务器上主站和 chat-server 会自动安装、构建并由 PM2 启动。

## 四、环境变量（必配）

- **本地**：项目根目录要有 `.env`（可复制 `.env.example` 再改）。用 Live Chat 时，根目录和 `services/chat-server` 的 `.env` 里 `CHAT_ADMIN_JWT_SECRET` 必须一致。
- **服务器**：在 `/root/website-new/.env` 配好 `DATABASE_URL`、`SESSION_SECRET`、`CHAT_SERVER_INTERNAL_URL=http://127.0.0.1:4000`、`CHAT_ADMIN_JWT_SECRET` 等，详见 `docs/LIVE-CHAT-上线检查.md`。

缺什么变量时，对照 `.env.example` 和文档补全即可。

## 五、后台登入不到

按下面顺序做一遍，多数情况能恢复登录。

1. **确认数据库有表**（本地 SQLite 时）  
   在项目根目录执行：
   ```powershell
   npm run migrate:sqlite
   ```
   若报错，先执行 `node scripts/prisma-generate-auto.cjs` 再试。

2. **创建管理员账号**  
   确保 `.env` 里有 `ADMIN_SEED_EMAIL` 和 `ADMIN_SEED_PASSWORD`（可参考上面「四、环境变量」或 `.env.example`），然后执行：
   ```powershell
   npm run seed:admin
   ```
   看到输出 `Admin seeded: xxx@xxx.com` 表示成功。

3. **用种子账号登录**  
   打开 http://localhost:3000/admin/login ，用 `.env` 里配置的 `ADMIN_SEED_EMAIL` 和 `ADMIN_SEED_PASSWORD` 登录（例如默认 `admin@example.com` / `change_me_123`）。

4. **仍不行时**  
   - 看浏览器控制台或网络里 `/api/admin/login` 的响应：若是 401，说明邮箱或密码错误；若是 500，多半是数据库未迁移或连接失败。  
   - 服务器部署时请确认 `.env` 里 `SESSION_SECRET` 已设置（不能为空），否则生产环境会报错。
