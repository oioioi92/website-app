# 图片转网址 / 上传图片 — 线上部署检查清单

线上使用「图片转网址」或主题/游戏 Logo 等上传功能时，按此清单做一次即可，避免重复排查。

## 1. 环境变量（必填其一）

在**线上服务器**的 `.env` 中至少配置一项（推荐与前台域名一致）：

```env
# 推荐：与前台域名一致，图片链接不暴露后台
NEXT_PUBLIC_FRONTEND_URL="https://admin1167.com"
NEXT_PUBLIC_UPLOAD_PUBLIC_URL="https://admin1167.com"
```

- **NEXT_PUBLIC_FRONTEND_URL**：侧栏「打开前台」、图片转网址页面拼完整链接时会用，必填。
- **NEXT_PUBLIC_UPLOAD_PUBLIC_URL**：上传接口返回的图片根地址，填了接口直接返回完整 URL。

## 2. 改完 .env 后必须重建并重启

- `NEXT_PUBLIC_*` 会在 **build 时** 打进前端包，改 .env 后必须重新执行 `npm run build`（或等同命令），再重启进程（如 `pm2 restart`）。
- 只重启不 rebuild，前端拿到的仍是旧值。

## 3. 图片由应用提供，不依赖 Nginx 静态目录

- 上传文件保存在 `public/uploads/`，访问走 **API**：`/api/public/uploads/...`。
- 旧链接 `/uploads/...` 已通过 `next.config.ts` 重写到上述 API，无需在 Nginx 里单独配静态路径。
- 只要前台、后台由**同一台 Next 应用**提供（同一 Nginx upstream），图片即可访问。

## 4. 可选：R2 云存储

- 若希望图片永久可用、不依赖本机磁盘（如多实例、无状态部署），请配置 R2（见 `.env.example` 中 R2_* 与 R2_PUBLIC_BASE_URL）。
- 配置 R2 后，上传会直接返回 R2 的 CDN 地址，无需再依赖本机 `public/uploads/`。

## 5. 验证

1. 后台 → 工具 → 图片转网址，上传一张图。
2. 复制得到的链接（应为 `https://前台域名/api/public/uploads/...` 或 R2 地址）。
3. 浏览器新标签打开该链接，应能正常显示图片。

---

**相关代码位置（便于以后排查）：**

- 上传 API 返回的 URL 逻辑：`app/api/admin/upload/image/route.ts`
- 公开访问根地址（服务端）：`lib/upload-public-url.ts`
- 提供图片的 API：`app/api/public/uploads/[[...path]]/route.ts`
- 前端拼完整链接：`components/admin/AdminImageToUrlClient.tsx`（用 `NEXT_PUBLIC_FRONTEND_URL` / `NEXT_PUBLIC_UPLOAD_PUBLIC_URL`）
