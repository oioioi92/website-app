# 上线前快速检查

部署前请按本清单逐项确认。详细说明与加固建议见《安全与加固建议.md》。

## 代码侧已实现

- **Session Cookie**：`app/api/admin/login/route.ts` 中已设置 `httpOnly: true`、`sameSite: "lax"`，生产环境 `secure: true`（`NODE_ENV === "production"`）。
- **后台 API**：`/api/admin/*` 均通过 `getAdminUserFromRequest` 校验会话，未登录返回 401。
- **登录限流**：后台登录 10 次/分钟/IP，降低暴力破解风险。

## 上线前必做

1. **环境变量**：生产环境配置 `SESSION_SECRET`、`CSRF_SECRET`、`DATABASE_URL` 等，且不提交到 Git。
2. **图片转网址/上传图片**：线上使用上传或「图片转网址」时，在 .env 中配置 `NEXT_PUBLIC_FRONTEND_URL`（及可选 `NEXT_PUBLIC_UPLOAD_PUBLIC_URL`）为前台域名，改后需重新 build 并重启。详见《图片转网址-部署检查清单.md》。
3. **HTTPS**：生产使用 HTTPS，确保 cookie `secure` 生效。
4. **依赖**：执行 `npm audit`，修复高危漏洞；生产使用 `npm ci`。
5. **chat-server**：不直接对公网暴露管理能力，仅通过 Next 内部 URL 调用。
6. **WhatsApp**：若启用 Webhook，配置 `WHATSAPP_APP_SECRET` 并验签。

完整检查项见《安全与加固建议.md》第 6 节检查清单。
