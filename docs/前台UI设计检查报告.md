# 前台 UI 设计检查报告

针对**电话版（移动端）**与**网站版（桌面端 lg+）**的合理性检查与改进建议。

---

## 一、电话版（你算喜欢，但怕有不合理）

### 1. 已做得比较合理的地方
- **布局**：顶栏（Logo + 品牌 + 合作方）+ 跑马灯 + 主内容 + 底部导航 + LiveChat 悬浮，层次清晰。
- **安全区域**：`pb-[max(7rem, ...)]`、`safe-area-inset-bottom` 已考虑刘海/横条，底部内容不会被遮挡。
- **底部导航**：固定底栏、当前页高亮、图标+文字、支持外链与站内链接，逻辑清晰。
- **主内容**：轮播 → Slot+LiveTx 一组 → 行动栏 → 游戏分类/列表，信息顺序合理。

### 2. 可能不合理或需注意的点

| 项目 | 说明 | 建议 |
|------|------|------|
| **顶部菜单按钮 ☰** | 目前仅占位，无下拉/侧栏，用户点击无反应。 | 要么接上真实菜单（导航、设置等），要么先隐藏或改为「返回」等有用操作，避免「假按钮」。 |
| **顶栏右侧「44」两个圆块** | 视觉上像数字角标，含义不明确（品牌 44？）。 | 若为品牌元素可加 `aria-hidden` 或 title；若可点击需有说明或链接。 |
| **跑马灯与主内容间距** | 跑马灯和首屏内容之间若过紧，会显得挤。 | 已用 `pb-[max(7rem,...)]` 留白；若仍觉得紧，可给主内容区加 `pt-2` 或 `pt-3`。 |
| **底部导航 5 个以上** | `theme.bottomNav.slice(0, 5)` 最多 5 个，若后台配了更多会被截断。 | 5 个对手机已够用；若将来要更多，需考虑折叠或「更多」入口，否则会挤。 |
| **LiveChatFab 与底部导航重叠** | 悬浮按钮在右下，底栏也在底部，小屏上可能重叠或误触。 | 当前 `bottom-[96px]`（约在底栏上方）一般够用；若机型异常可再微调或加「收起」态。 |
| **游戏分类横向滚动** | 分类多时横向滑动的可发现性依赖「可滑动」视觉。 | 若列表较长，可在右侧留一点「被裁切」的下一项，或加淡出边，提示可滑动。 |

### 3. 小结（电话版）
- 整体结构、安全区、导航逻辑都合理，没有明显「错」的设计。
- 主要风险是**顶栏 ☰ 无功能**会让人感觉不合理；其余多为细节优化。

## 二、网站版（你完全不喜欢 — 问题与改进方向）

### 1. 当前桌面端的主要问题

| 问题 | 具体表现 | 影响 |
|------|----------|------|
| **两套桌面头不一致** | `DesktopHeader`（旧）与 `DesktopHeaderV3`（新）并存，用 `useV3Layout` 切换。旧版有「报表/分析/活动/社区」等导航但链到 `#` 锚点，多数无对应区块；V3 只有 Logo+品牌+语言+「44」圆块，左侧 ☰ 无功能。 | 体验割裂；导航要么无效要么缺失。 |
| **桌面主导航缺失** | V3 顶栏没有「首页 / 优惠 / 游戏 / 客服」等主 nav；旧版有 nav 但指向 `#results`、`#analyzer` 等，页上未必有对应 id。 | 用户不知道去哪找优惠、游戏、客服。 |
| **右侧边栏像占位** | 固定「Balance: RM 0.00」「Winover: RM 0.00」、Surrender 链接、TRUSTED SITE 竖排字等，像模板占位，未与登录态/真实数据打通。 | 显得假、不专业。 |
| **主内容区信息堆砌** | 轮播 + 中间槽图 + LiveTx + 钱包栏 + Referral 横幅 + ModuleGrid + ReportsAndPromos + Games Preview（Coming soon），一列下来很长，层次不够清晰。 | 重点不突出，像「把所有模块都摆上去」。 |
| **品牌/文案偏模板** | 「PERODUA 44」、绿/红 44 圆块、REFERRAL COMMISSION 20%、JOIN NOW 等，若与真实品牌不符会显得生硬。 | 缺乏「这是我们网站」的专属感。 |
| **Hero（非 V3）与业务脱节** | 旧版 Hero 写「Top Picks & Live Status」「Get data-driven picks」等，更像数据/博彩分析站，和「优惠+存款+客服」主流程不一致。 | 首屏价值不清晰。 |
| **Footer 较弱** | 仅社交+法律链接，且部分链到 `#`；没有站点简介、客服入口、支付方式等。 | 桌面用户期望的「完整网站感」不足。 |

### 2. 改进方向（网站版）

- **统一一套桌面顶栏**
  - 只保留一个 header（建议以 V3 为基础），并加上**真实主导航**：首页、优惠/活动、游戏（或跳转）、客服/联系等，链接到实际路由（如 `/bonus`、`/promotion`、`/chat`）。
  - ☰ 要么做成侧栏/下拉菜单（导航、语言、客服），要么移除，避免无效按钮。

- **右侧栏二选一**
  - **方案 A**：改成「登录/注册 + 登录后的余额/快捷操作」，与后端/登录态打通，至少余额为 0 时也显示「已登录」状态。
  - **方案 B**：若暂时不做登录态，收窄为「登录 / 注册 / 客服」三个主按钮 + 客服链接，去掉 Balance/Winover/Surrender/TRUSTED SITE 等占位，避免「假数据」。

- **首屏重点**
  - 第一屏：品牌 + 主 nav + 1 个主轮播（或 1 个主 CTA）+ 登录/注册/存款入口，其余（LiveTx、推荐、游戏预览）下移或收在「更多」里。
  - 让「来干什么」在 2 秒内能看懂：注册、存款、看优惠、找客服。

- **视觉与品牌**
  - 用主题/后台配置：主色、Logo、站点名、一句 Slogan，替换掉硬编码的 PERODUA 44、固定绿红等，让网站版和电话版都像「同一品牌」但桌面更舒展。

- **Footer**
  - 增加一段 1～2 行站点简介、客服/WhatsApp/Telegram、支付方式说明（若适用）、备案号等，链接都指向真实页面或外链。

### 3. 小结（网站版）
- 当前问题主要是：**导航无效或缺失、右侧栏像占位、主内容堆砌、品牌/文案模板化**。
- 按上面「统一顶栏 + 真实导航」「右侧栏简化或对接登录」「首屏突出重点」「品牌可配置」「Footer 补全」改一版，网站版会从「不喜欢」明显改善。

## 三. 建议的下一步（可交 Cursor 实现）

1. **电话版**  
   - 顶栏 ☰：接上简单菜单（至少：首页、优惠、客服/联系）或暂时隐藏。

2. **网站版**  
   - 以 V3 为基础，加**主导航**（首页、优惠、游戏、客服），链接到 `/`、`/bonus`、`/promotion`、`/chat` 等。  
   - 右侧栏：改为「登录/注册/客服」+ 可选「登录后显示余额」；移除或替换占位文案（Balance 0、Surrender、TRUSTED SITE）。  
   - 首屏：保留轮播 + LiveTx + 钱包栏，把 ModuleGrid / ReportsAndPromos / Games 收成「下方区块」或 Tab，避免首屏过长。  
   - Footer：补一句站点说明 + 客服入口 + 法律链接，避免全部 `#`。
