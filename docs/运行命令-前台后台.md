# 每次运行：前台 + 后台 命令

**说明**：前台和后台的链接（localhost:3000 和 /admin/login）**必须先把开发服务跑起来才能打开**，否则浏览器会打不开。下面就是“启动服务”要用的命令。

---

## 推荐：一键启动并自动打开前台、后台

在项目**根目录**打开 PowerShell，执行：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\dev-sqlite-and-open.ps1
```

会做的事：
1. 若还没有 `.env`，自动用 SQLite 配置并初始化数据库
2. 在新窗口启动开发服务器
3. 等服务就绪后，自动用默认浏览器打开：
   - **前台**：<http://localhost:3000/>
   - **后台**：<http://localhost:3000/admin/login>

之后每次只要执行这一条命令即可。

---

## 手动启动（不自动打开浏览器）

若只想启动服务，自己再手动在浏览器里输入地址，可以用下面两种之一。

### 方式 A：无数据库 / 本地 SQLite

在项目**根目录**执行：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\dev-sqlite.ps1
```

服务起来后，**再**在浏览器中打开：
- **前台**：<http://localhost:3000/>
- **后台**：<http://localhost:3000/admin/login>

### 方式 B：已配置 Postgres（.env 里 DATABASE_URL 为 postgres）

在项目**根目录**执行：

```powershell
npm run dev
```

服务起来后，**再**在浏览器中打开：
- **前台**：<http://localhost:3000/>
- **后台**：<http://localhost:3000/admin/login>

---

## 若终端报错 `npm 不是内部或外部命令`

先执行（把 Node 加入当前会话的 PATH）：

```powershell
$env:Path = "C:\Program Files\nodejs;" + $env:Path
```

然后再执行上面的「一键启动」或「手动启动」命令。
