## 二、报表 slug 与 API 映射（lib/backoffice/report-center-config.ts）

- **bonus** → reportKey `bonus-cost`（红利成本汇总）
- **bank** → reportKey `reconciliation`（对账）
- **bonus-cost-summary** → `bonus-cost`
- 其余 slug 与 reportKey 一致或走 ledger-transactions 等

   - `SLUG_TO_REPORT_KEY["bonus"] = "bonus-cost"`  
   - 三语 `admin.reportTitle.bonus`、`SLUG_TO_TITLE["bonus"]` 已补

2. **/admin/reports/bank**  
   - `REPORT_CENTER_SLUGS` 增加 `"bank"`

   - 三语 `admin.reportTitle.bank`、`SLUG_TO_TITLE["bank"]` 已补

3. **Promotions 空状态**  
   - 两处「暂无优惠活动」改为 `t("admin.promotionsList.noPromotions")`  
   - ZH/EN/MS 已补 `noPromotions` 文案

## 四、报表中心 REPORT_CENTER_SLUGS 覆盖情况

以下 slug 均走 `ReportTableFromApi`（或专项组件），无 404：

- all-transactions, deposit-report, withdraw-report  
- hourly-sales, winloss-by-game, winloss-by-player  
- **bonus**, bonus-transactions, bonus-cost, bonus-cost-summary  
- transfer-report  
- gateway-search, reconciliation, **bank**  
- user-kpi  

特殊处理：

- **turnover-by-game**：占位页 + 跳转 winloss-by-game（按产品设计保留）
- **transactions-detail**：ReportTransactionsDetail 组件
- **daily-summary**：ReportDailySummary 组件

## 五、API reportKey 支持（app/api/admin/reports/query/[reportKey]）

已实现并供前端使用的 reportKey：  
ledger-transactions, hourly-sales, winloss-by-game, winloss-by-player, bonus-cost, user-kpi, gateway-search, reconciliation 等。  
`bonus`、`bank` 不直接作为 reportKey，通过 slug 映射到 `bonus-cost`、`reconciliation`。
